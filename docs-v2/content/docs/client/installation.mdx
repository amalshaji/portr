### Install the client

Using the install script:

```shell
curl -sSf https://install.portr.dev | sh
```

Or install the client using homebrew:

```shell
brew install amalshaji/taps/portr
```

You can also download the client binary from [github releases](https://github.com/amalshaji/portr/releases).

### Configure authentication

Once the download/install is complete, login to the portr admin dashboard and copy the setup command from the overview page. It'll look like this:

```bash
portr auth set --token {your_token} --remote {your_domain}
```

The command downloads a client config into `~/.portr/config.yaml`. The server's
`PORTR_TRANSPORT` setting decides which transport appears in generated client
configs. SSH is the default:

```yaml
server_url: example.com
transport: ssh
ssh_url: example.com:2222
secret_key: portr_xxx
```

If the server is configured with `PORTR_TRANSPORT=websocket`, generated configs
use the WebSocket tunnel endpoint instead:

```yaml
server_url: example.com
transport: websocket
ws_url: example.com
secret_key: portr_xxx
```

Use `server_url` for admin API calls. Use `ssh_url` or `ws_url` for
client-to-server tunnel connections, depending on the generated `transport`.

### Verify installation

For more details and to verify everything is working, run:

```bash
portr --help
```

<Callout type="info">
  Make sure you have access to a Portr server before setting up the client. If you need to set up your own server, check out the [server setup guide](/docs/server).
</Callout>
